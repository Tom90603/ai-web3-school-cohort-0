## 一、区块链的物理本质

区块链是一条链式结构的数据记录（账本），由全球分布的节点共同维护。

- **链**：每条链有自己的节点网络、共识规则、原生代币。不同链之间相互独立。
- **节点（记账员）**：运行全节点软件的计算机，保存完整历史数据。全节点存储量随链上活动增长，以太坊全节点当前约 1TB，未来有持续增长的趋势。
- **共识机制**：节点间就"哪笔交易有效"达成一致的规则。比特币采用工作量证明（PoW），以太坊采用权益证明（PoS）。

### 知名区块链及定位

| 链 | 原生代币 | 定位 | 关键特征 |
|----|---------|------|---------|
| Bitcoin | BTC | 数字黄金/价值存储 | 总量恒定 2100 万，仅支持简单转账 |
| Ethereum | ETH | 可编程的世界计算机 | 支持智能合约，开发者与生态最大 |
| Solana | SOL | 高性能并行链 | 低费用，高吞吐量 |
| Polygon | MATIC | 以太坊侧链 | 兼容以太坊工具链，费用较低 |

### 存储极限问题

区块链全节点存在物理存储上限。解决方案包括：
- **状态裁剪**：只保留当前状态快照，丢弃早期历史
- **分片**：不同节点分别存储不同部分的数据
- **L2 Rollup**：详细交易数据存在 L2，主链只存最终结果哈希
- **无状态客户端**：节点不存储完整状态，只验证交易+状态证明

该问题的实际瓶颈不在总存储量，而在"存储成本谁来承担"——如果只有大机构能运行全节点，会重新中心化，与去中心化初衷矛盾。

## 二、代币的分类体系

### 2.1 原生代币

链本身的货币，用于支付手续费和激励节点。

**BTC**：总量恒定 2100 万，不可增发。角色定位为价值存储。

**ETH**：非固定供应。2021 年 EIP-1559 升级引入销毁机制——每笔交易的基础费被销毁。ETH 总供应量由"新铸造量"与"销毁量"的动态差值决定，网络活跃时可能通缩，冷清时可能通胀。

### 2.2 链上发行的代币（Token）

在已有链上通过智能合约发行的代币。本质是链上合约数据库中的一条记录，不拥有自己的独立网络。

| 类别 | 代表 | 用途 |
|------|------|------|
| 稳定币 | USDT, USDC | 锚定法定货币价值（如 1 USDT ≈ 1 USD），用于交易媒介和储值 |
| 治理代币 | UNI, AAVE | 持有者对协议未来决策（升级、费率等）投票 |
| 功能代币 | LINK, FIL | 在特定生态中作为服务支付的必需代币（如 Chainlink 节点质押） |

**稳定币机制对比**

| 特征 | USDT（Tether） | USDC（Circle） |
|------|--------------|---------------|
| 发行方 | Bitfinex 关联方，香港注册 | Circle，美国持牌公司 |
| 审计透明度 | 无完整公开审计 | 每月由 Grant Thornton 审计并公开报告 |
| 合规 | 曾受多国监管调查 | 受美国纽约州 DFS 等机构监管 |
| 市场份额 | ~70% | ~20% |

USDT 是有争议的——准备金透明度长期受质疑，且曾因 Bitfinex 挪用准备金被纽约总检察长调查。但市场因其先发优势和流动性深度，仍将其作为主要稳定币使用。USDC 在机构市场（对冲基金、做市商等需要合规的参与者）中占据主流。

2023 年硅谷银行事件中，USDC 的 33 亿美元准备金被锁定在硅谷银行内，导致 USDC 一度脱锚至 0.88 美元，后被救助恢复锚定。该事件说明：即便合规的稳定币也存在准备金所在银行的信用风险，但其抵押物的可追溯性本身也是合规优于竞品的体现。

### 2.3 发币的机制与流程

在以太坊上发行一个 ERC-20 代币的标准流程：

1. **编写合约**：使用标准合约模板（如 OpenZeppelin），定义代币名称、符号、总供应量
2. **部署到链上**：通过开发框架（Remix、Hardhat、Foundry）将合约部署至目标链
3. **充入流动性（可选但必要）**：在去中心化交易所（如 Uniswap）创建流动性池，使代币可交易
4. **获得认可以及分发**：在区块浏览器验证合约源码、申请收录到行情网站、社区宣传、寻找交易所上币

该流程的技术门槛较低，有区块链开发经验者在 1 小时内即可完成。低门槛的正反面：一方面降低了创新实验成本，另一方面也为欺诈创造了条件。

### 2.4 哪些代币不是空气

具有实际用途的代币通常满足以下条件之一：
- 持有代币可对真实产品进行治理（如 UNI 对应 Uniswap——全球最大 DEX）
- 代币作为系统的安全抵押品（如 LINK——节点质押后若提供假数据会被罚没）
- 持有代币可享受协议的实际收益或折扣（如 AAVE——折抵借贷费用、参与安全模块）

三类代表：

- **UNI（Uniswap）**：去中心化交易所 Uniswap 的治理代币。Uniswap 日交易量约 20 亿美元，UNI 持有者可投票决定协议参数。类比：苹果股票——不意味着拥有一台 iPhone，但代表对公司的部分话语权。
- **LINK（Chainlink）**：预言机网络 Chainlink 的质押代币。节点需质押 LINK 作为诚信押金，虚假数据会被罚没。LINK 在该生态中是保障数据可靠性的经济工具。
- **AAVE（Aave）**：借贷协议 Aave 的治理代币。Aave 管理链上存款超百亿美元，AAVE 持有者可参与治理，在安全模块中质押可获得协议收益分成。

### 2.5 跨链代币

同一个代币通过"跨链桥"在多个链上发行。不同链上的版本是各自独立的合约记录，通过跨链桥保持价值锚定。以 USDC 为例，其同时在 Ethereum、Solana、Polygon 等链上存在独立版本，用户跨链转移时需通过跨链桥锁定源链版本的 USDC 并在目标链铸造等量版本。

## 三、DeFi（去中心化金融）

用智能合约替代传统金融中介（银行、券商、交易所）的金融操作。

### 3.1 关键协议

| 功能 | 传统金融 | DeFi 代表 |
|------|---------|-----------|
| 交易/兑换 | 币安、Coinbase | Uniswap、Sushiswap |
| 借贷 | 银行 | Aave、Compound |
| 稳定币 | USDT/USDC | MakerDAO (DAI) |
| 衍生品 | CME | dYdX、Synthetix |

### 3.2 Uniswap 工作原理：AMM（自动做市商）

核心公式：**x × y = k**

- x = 池子里代币A的数量
- y = 池子里代币B的数量
- k = 常数

用户用代币A兑换代币B时：向池子注入代币A → 池子按恒定乘积公式计算出可兑出的代币B数量 → 池子比例改变，价格自动更新 → 0.3%手续费归流动性提供者。

与传统订单簿交易所的对比：

| 特征 | 订单簿（如币安） | AMM（如 Uniswap） |
|------|--------------|----------------|
| 价格发现 | 买卖双方挂单撮合 | 数学公式自动定价 |
| 流程复杂度 | 需等对手单，深度不足时滑点大 | 池子固定，有足够的流动性提供时执行确定 |
| 对手方 | 需找到匹配的买方/卖方 | 池子就是对手方，总有流动性 |

### 3.3 当前市场规模

- DeFi 总锁仓量（TVL）：约 800-1000 亿美元
- Uniswap 日交易量：约 20 亿美元
- Aave 活跃借贷：约 50 亿美元
- 稳定币总市值：约 1500 亿美元

### 3.4 DeFi 与中心化金融的关系：目前替代了多少

**技术上能做到的事：**
- 跨境汇款：USDT/USDC 已成为部分发展中国家（如阿根廷通胀环境下）的生存刚需
- 链上借贷：超额抵押，无需审批和信用分，由市场利率自动调节
- 去中心化交易：Uniswap 交易量与 Coinbase 等中心化交易所处于同一量级

**中心化金融仍不可替代的功能：**
- 基于信用的无抵押贷款（DeFi 目前只能做超额抵押借贷）
- 大规模资金的保险保障和争议处理
- 与其他金融系统的互操作（税务对接、工资代发等）

### 3.5 碎片化现状

传统银行的体验：一个 App 覆盖存款、借贷、转账、理财。

Web3 当前状态：转账用钱包 App → 兑换去 DEX（Uniswap）→ 借贷去协议（Aave）→ 资产管理用聚合器（Zapper）——各步骤由不同团队、不同协议完成，用户需手动搭桥。

这不是产品经理怠惰的结果，而是源于 Web3 的底层设计哲学：各协议由独立团队开发部署，没有中央协调者统一功能入口。

## 四、预言机——Chainlink 的工作原理

### 4.1 问题背景

智能合约运行在封闭的区块链环境中，无法直接访问链外数据（如实时价格、天气、比赛结果），需要预言机作为桥梁。

### 4.2 工作流程

```
用户合约 → 请求数据 → Chainlink 聚合合约 
    → 随机选取节点子集（21-30 个）
    → 每个节点独立从多个数据源（CoinGecko、币安、Coinbase 等）拉取数据
    → 各节点提交数据回到聚合合约
    → 聚合合约去除最高值和最低值，取中位数
    → 存储结果供用户合约读取
```

### 4.3 信任机制：经济激励

节点运营者需质押 LINK 代币作为保证金。提供准确数据可获得代币奖励；若提供偏离中位数过远的数据，质押的代币会被罚没（Slashing）。

用户不信任任何一个节点个体，而是信任"节点不希望自己的钱被罚没"这一经济动机。这也是预言机领域公认的可行方案。

## 五、用户体验相关议题

### 5.1 协议信任 vs 产品信任

| 层面 | 保证机制 | 能解决的问题 |
|------|---------|-------------|
| 协议信任（代码/机制层） | 共识机制 + 开源智能合约 | "代码的执行不受人为干预" |
| 产品信任（人为层） | 团队背景 / 审计报告 / 透明度披露 | "这个项目方不会套现跑路" |

区块链的共识机制只保证前者。后者需要额外的信息。行业中大量欺诈行为发生在"产品信任"这一层面——项目方从不需要通过篡改合约来作恶，只需在高位抛售自己囤积的代币即可。目前尚无单一机制能完全解决这一层面的信任问题。

### 5.2 语言与信息壁垒

- 主流 Web3 产品（钱包、DEX、分析工具）以英语为默认语言，核心术语（Gas Fee, Swap, Slippage, AMM）缺乏中文原生对应物
- 教育材料以英文为主，学习曲线对非英语用户较高
- 产品界面极少考虑非金融背景用户的使用习惯

由于加密货币的经济回报在一定程度上覆盖了使用门槛，英语/金融术语障碍尚未成为行业优先解决的产品问题。

### 5.3 解决方案方向

**账户抽象（Account Abstraction）**
将钱包从"单一私钥签名"升级为可编程的智能合约钱包。支持的场景包括：
- 多签授权：N 人中的 M 人同意才能操作
- 社交恢复：丢失私钥可通过预设的好友恢复
- 会话密钥：授权特定应用在一段时间内代持操作
- 批量交易：一次签名授权多步操作连续执行

**前端聚合器**：在界面层屏蔽底层协议差异。用户只感知一个入口，后台自动路由至最优协议。代表产品包括 Zapper、Zerion、DeBank。该方案的局限在于聚合器本身是一个中心化入口，用户需要信任聚合器合约的安全性。

## 六、AI × Web3 交叉叙事的现状

该方向是将两个领域结合的行业叙事，目前主要聚焦在三条技术路线：

### 6.1 Agent 经济自主权（链上钱包 + 自动支付）

AI Agent 在区块链上拥有自己的智能合约钱包，能够自主支付 API 调用费用、与其它 Agent 进行链上结算、以及自动分配订阅收入。

该方案解决的问题是：Agent 在传统体系下无法拥有银行账户，无法自主支付。链上钱包充当 Agent 的"链上银行账户"。资金来源和每笔支出均在链上可查，且可通过智能合约设置花费规则（单笔上限、白名单目标地址等）。

目前的核心争议：User Agent 需要自主支付是产品侧的真实需求还是技术侧的叙事需求——用户关心的"Agent 的服务质量"与"Agent 如何付钱给服务方"是两个不同的问题。

### 6.2 训练数据确权与交易

设想：创作者将作品通过智能合约注册到链上，记录所有权和时间戳。当 AI 模型使用该作品训练时，自动追踪使用量并分配收入。

该设想的挑战：目前尚无技术手段可精确追踪"特定数据是否被用于训练某模型"；确定分发比例的合理性无法被验证；链上分配收入的金额可能极低以至于无法覆盖手续费。

### 6.3 去中心化 AI 推理网络

设想：由全球各地贡献的 GPU 算力组成去中心化的模型推理网络，不依赖 OpenAI 等中心化服务商。

面临的困难：推理速度无法满足实时对话需求；模型输出的一致性缺乏有效验证手段；分布式推理的算力利用率与单位效率不及中心化数据中心。

## 七、关键概念索引

| 术语 | 定义 |
|------|------|
| AMM（Automated Market Maker） | 自动做市商，通过固定公式（如 x*y=k）为交易对自动定价 |
| TVL（Total Value Locked） | 锁仓总价值，衡量 DeFi 协议中存放资产的总量 |
| Gas Fee | 链上交易的网络手续费，由基础费（销毁）+ 小费组成 |
| L1 / L2 | 第一层（主链）/ 第二层（通过主链保证安全、费用更低的子链） |
| EIP-1559 | 以太坊协议升级，引入基础费销毁机制，ETH 因此可能通缩 |
| Slashing | 罚没机制，节点违规时损失质押的代币 |
| Slippage | 滑点，交易执行价格与预期价格的差异 |
| Liquidity Pool | 流动性池，用户存入代币为交易提供流动性的智能合约 |
| Cross-chain Bridge | 跨链桥，使代币在不同区块链之间转移的协议 |
| Account Abstraction | 账户抽象，将钱包由私钥对升级为可编程合约钱包的技术方案 |
