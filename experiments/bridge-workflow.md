# 🌉 Bridge 层工作流链路图

> 产出时间：2026-05-27 | 学员：Tom90603

## Agent 链上工作流

```
用户目标
    │
    ▼
┌──────────────────────────────────────────┐
│  ① Chain-aware Context（链感知上下文）       │
│  ─ 读取 chain id、合约地址、ABI、余额、       │
│    授权、交易历史、区块数据                   │
│  ─ 链上状态带来源：区块号、交易哈希、          │
│    explorer link                           │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│  ② Web3 Tool Use（工具调用）                │
│  ─ 读写分离：Contract Read ≠ Write         │
│  ─ 只读：RPC Tool / Contract Read          │
│  ─ 写链：Contract Write / Wallet Tool      │
│  ─ 检查：权限 + 额度 + 模拟(simulation)     │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│  ③ Agent Workflow（智能体工作流）             │
│  ─ Task Graph：目标拆成节点                 │
│  ─ State Machine：明确阶段状态（draft →     │
│    context_loaded → plan_ready → waiting_  │
│    user_confirmation → submitted → done）   │
│  ─ Trace：每一步都有记录                    │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│  ④ Human-in-the-loop（人工复核）            │
│  ─ 只读分析 → 自动执行                      │
│  ├ 交易草稿 → 自动生成                      │
│  ├ 小额白名单 → Session Key                │
│  ├ 高风险交易 → 人工确认                    │
│  └ 超出范围 → 直接拒绝                      │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│  ⑤ 签名上链                                 │
│  ─ 用户签名 / Session Key / 多签            │
│  ─ 提交交易 → 等待确认                      │
│  ─ 失败时：判断是否已广播 → 停止/重试          │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│  ⑥ 验证结果 + 记录                          │
│  ─ 交易哈希 + explorer link                │
│  ─ 写入 Trace                              │
│  ─ 回应用户 + 更新上下文                    │
└──────────────────────────────────────────┘
```
